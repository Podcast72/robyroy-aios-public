# Roadmap

The public roadmap for this repository is intentionally modest.
It focuses on increasing clarity and proof quality rather than adding speculative claims.

## Near-term areas

1. Expand the public proof set with additional read-only and fail-closed runtime cases.
2. Publish a more detailed public schema for audit events and trace interpretation.
3. Add more boundary documentation for compat and support surfaces without promoting them to the official backbone.
4. Improve the public tests so that architectural regressions are easier to detect from the documentation and examples alone.

## What is intentionally not on this roadmap

- publishing the full private operational repository
- broad compatibility claims without matching proof artifacts
- marketing-style benchmark narratives without technical support
